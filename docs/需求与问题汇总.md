## 🔴 当前问题（按优先级）

### B1. 无法暂停 / 暂停切换卡顿（CRITICAL，复发）
- 现象：播放后点暂停（或空格）无法停止，"只能播放"；暂停切换时卡顿。
- **首要嫌疑（待验证修复）**：`TimelinePlaybackLayer` 卸载时，per-render 的 ref 回调 `els.current.delete(id)` 可能在 effect cleanup 之前执行，导致 cleanup 里 `for (el of els.current.values()) el.pause()` 拿到空 Map → `<audio>`/`<video>` 元素**从 DOM 移除后仍继续播放**（浏览器不会自动暂停被移除的媒体元素）。
  - 修复方向：给 TimelinePlayback 根 div 加 `containerRef`，cleanup 改为 `containerRef.current?.querySelectorAll('video,audio').forEach(el=>el.pause())`（卸载时 DOM 仍在，可靠）。
- **卡顿嫌疑**：暂停瞬间 `useTimelineFrame` 重新启用 → 触发 `composite_frame`（ffmpeg 解码+PNG）同步开销 → UI 卡。可考虑暂停后延迟/异步取合成帧，或复用最后一帧。
- 文件：`web/src/components/preview/TimelinePlaybackLayer.tsx`、`Preview.tsx`、`useTimelineFrame.ts`、`hooks/usePlaybackTicker.ts`。

### B2. 音频波形不渲染（HIGH）
- 现象：有音乐/音频，但 A1 轨是纯色块，看不到波形。
- 进展：**后端 `get_waveform` 及端口已完成接入**，但实测仍不显示（被前端 catch 吞掉 → 显示 faint band）。
- 嫌疑：clip 的 mediaRef 解析路径问题；或 symphonia 无法解码该音/视频容器的音轨；或音频是视频的 linked-audio（mediaRef 指向视频文件）。
- 排查：在 `get_waveform` 失败时加日志；确认 clip.mediaType==='audio' 且 mediaRef→可解析 path；真机/或写 Rust 集成测试用真实音频文件跑 `engine.waveform`。
- 文件：`src-tauri/src/media.rs` `get_waveform`、`web/src/components/timeline/TimelineContainer.tsx`(加载)、`clipRenderer.ts` `drawWaveform`。

### B3. 丢失素材 → 全红且不恢复（HIGH）
- 现象：选定素材路径不可访问时显示"丢失素材"；重新选择该路径后提示，但**片段全部显示红色，左侧媒体库也红**，未恢复正常。
- 排查：媒体"丢失/重链接(relink)"逻辑——路径不可达时的状态标记 + 重新指定路径后的清除红色状态。对照上游 `EditorViewModel+Relink.swift`。
- 文件：`src-tauri/src/media.rs`、`web/src/store/mediaStore.ts`、媒体面板渲染、clip 渲染的 missing 状态。

## 🟦 新功能需求

### F1. 触控板手势（剪映式）
- 双指滑动触控板 → 直接移动时间线位置（移动播放头/横向滚动）。
- 双指捏合 → 缩放时间线。
- 当前是 Option/Cmd+滚轮缩放（`TimelineContainer.tsx` onWheel）。需改为：裸双指滚动=平移、捏合(ctrlKey on wheel = pinch)=缩放，贴近剪映操作逻辑。
